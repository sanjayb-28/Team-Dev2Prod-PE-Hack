# Security Policy

## Reporting A Vulnerability

If you believe you have found a security issue in Dev2Prod, please do not post sensitive details in a public issue.

Instead:

1. contact the repository maintainer privately through GitHub
2. include a clear description of the issue
3. include steps to reproduce, impact, and any suggested mitigation if available

## What To Include

Helpful reports usually include:

- affected component or route
- reproduction steps
- expected versus actual behavior
- scope and impact
- whether the issue is configuration-specific or general

## Disclosure

Please allow time for triage and remediation before public disclosure.
